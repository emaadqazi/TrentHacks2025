# Comprehensive Resume Evaluation Criteria
## Industry-Leading Standards for AI Resume Feedback

---

## SCORING FRAMEWORK
**Target Score:** 80%+ for strong candidacy
**Scale:** 1-100 or 1-4 rating per category
- 90-100%: Excellent match - highly likely to advance
- 80-89%: Good match - meets most criteria
- 70-79%: Fair match - missing some key elements
- Below 70%: Weak match - significant gaps

---

## 1. ATS COMPATIBILITY & FORMATTING (Weight: 20%)

### File Format & Readability
- ✅ PDF or DOCX format (PDF preferred from Word/Google Docs)
- ✅ Text is highlightable (not scanned image)
- ✅ No tables, columns, or text boxes
- ✅ No headers/footers with critical information
- ❌ No graphics, images, logos, or icons
- ❌ No unusual fonts (stick to Arial, Calibri, Georgia, Times New Roman)

### Section Headers (Must be standard)
- "Work Experience" or "Professional Experience" (not "My Journey")
- "Education" (not "Academic Background")
- "Skills" or "Technical Skills" (not "What I Know")
- "Projects" (not "Cool Stuff I Built")
- Avoid creative naming that ATS won't recognize

### Layout & Structure
- Single column layout
- Consistent date formats (MM/YYYY throughout)
- Standard bullet points (•) - no arrows, checkmarks, or custom symbols
- Left-aligned text preferred
- Clear visual hierarchy
- White space for readability
- 1-2 pages maximum (1 page for <5 years experience)

### Parsing Score
- Resume can be correctly parsed by ATS
- All sections properly identified
- Dates, job titles, companies extracted correctly
- No information lost in conversion

---

## 2. KEYWORD OPTIMIZATION (Weight: 25%)

### Job Description Matching
- 70-85% keyword match with target job description (not 100% - avoid keyword stuffing)
- Hard skills from job description present
- Soft skills from job description included
- Industry-specific terminology used
- Technology/tool names spelled correctly (JavaScript not Javascript)

### Keyword Placement Strategy
- Keywords in Skills section for quick scanning
- Keywords naturally integrated in experience bullets
- Both acronyms AND full terms used (e.g., "UI/UX" and "User Interface/User Experience")
- Action verbs from job description incorporated

### Avoid
- Keyword stuffing (repeating same keyword unnaturally)
- Irrelevant buzzwords ("synergy", "think outside the box")
- Generic filler words without substance
- Keywords without context

### Recommended Action Verbs
**Leadership:** Led, Directed, Managed, Coordinated, Spearheaded, Orchestrated
**Achievement:** Achieved, Delivered, Exceeded, Launched, Drove, Generated
**Technical:** Developed, Engineered, Architected, Designed, Implemented, Built
**Improvement:** Optimized, Enhanced, Improved, Streamlined, Reduced, Increased
**Collaboration:** Collaborated, Partnered, Facilitated, Coordinated, Aligned

---

## 3. CONTENT QUALITY & IMPACT (Weight: 30%)

### Experience Section - The STAR Method
Each bullet should follow: **Action Verb + Task + Result/Impact**

**Formula:** [Strong Action Verb] + [What you did] + [Technologies/Methods] + [Quantifiable Result]

**Example:** 
❌ Bad: "Responsible for improving website performance"
✅ Good: "Optimized website load times by 40% through code refactoring and CDN implementation, improving user retention by 25%"

### Quantifiable Achievements (CRITICAL)
- Include numbers, percentages, dollar amounts, time saved
- Scale/scope indicators (team size, user base, data volume)
- Before/after comparisons
- Business impact metrics

**Examples:**
- "Increased revenue by $2M"
- "Reduced load time from 5s to 1.2s (76% improvement)"
- "Led team of 8 engineers"
- "Processed 10M+ daily transactions"
- "Deployed to 500K+ users"
- "Reduced bug rate by 35%"

### Impact-Focused Language
- Focus on outcomes, not just responsibilities
- What did you achieve, not what you were "responsible for"
- Active voice (not passive)
- Present tense for current role, past tense for previous roles

### Conciseness & Clarity
- 3-5 bullets per job (prioritize most impactful)
- Each bullet 1-2 lines maximum
- No paragraphs - only bullet points
- Every word must add value
- Remove redundant information

---

## 4. TECHNICAL SKILLS SECTION (Weight: 15%)

### For Tech Roles - Must Include:
**Programming Languages:** 
- List proficiency level if impressive (e.g., "Python (Expert, 10K+ lines)")
- Only include languages relevant to target role

**Frameworks & Technologies:**
- Group by category (Frontend, Backend, Database, Cloud, etc.)
- Include version numbers if specific to job
- Prioritize technologies from job description

**Tools & Platforms:**
- Development tools (Git, Docker, Jenkins, etc.)
- Cloud platforms (AWS, GCP, Azure)
- Relevant software/systems

### Skills Section Format
```
Languages: Python, JavaScript, Java, C++
Frameworks: React, Node.js, Django, Spring Boot
Databases: PostgreSQL, MongoDB, Redis
Cloud: AWS (EC2, S3, Lambda), Docker, Kubernetes
Tools: Git, Jenkins, JIRA, Postman
```

### Avoid
- Rating skills with bars/stars (unprofessional, hard to quantify)
- Including every technology ever touched
- Listing soft skills in technical skills section

---

## 5. PROJECTS SECTION (Weight: 10% for experienced, 20% for new grads)

### Project Structure
**Format:** [Project Name] (linked to GitHub/live demo if possible)
- Brief description (1 line)
- Technologies used
- Quantifiable impact or scope
- Your specific contribution (if team project)

**Example:**
```
InvestorsArena (github.com/username/investorsarena) | React, TypeScript, Firebase
Gamified stock market learning platform with 5K+ registered users
• Architected real-time trading system handling 1000+ concurrent users
• Implemented leaderboard algorithm reducing query time by 60%
• Integrated third-party stock APIs with 99.9% uptime
```

### Project Quality Indicators
- Links to GitHub/portfolio (active requirement)
- Real-world usage/impact metrics
- Technical complexity demonstrated
- Problem-solving approach evident
- 2-4 projects recommended
- Most recent/relevant projects first

---

## 6. EDUCATION SECTION (Weight: 5-10%)

### Required Information
- Degree type and major
- University name
- Graduation date (MM/YYYY) or expected graduation
- GPA if > 3.5/4.0 or > 3.7/4.0 for top companies
- Relevant coursework (if new grad, < 3 courses listed)

### For New Graduates
- Include relevant thesis/capstone project
- Academic achievements (Dean's List, scholarships)
- Relevant coursework (3-5 courses maximum)

### For Experienced Professionals (5+ years)
- Minimal education section
- No GPA needed
- No relevant coursework
- Focus space on experience instead

---

## 7. CONTACT INFORMATION & HEADER (Weight: 5%)

### Must Include
- Full name (prominent at top)
- Phone number
- Professional email (firstname.lastname@domain.com)
- LinkedIn URL (customized)
- GitHub/Portfolio URL (for tech roles)
- Location (City, State) - optional but helpful

### Optional/Strategic
- Personal website
- Coding platform profiles (LeetCode, HackerRank) with rankings
- Notable open-source contributions

### Avoid
- Unprofessional email addresses (coolcoder69@...)
- Full street address (privacy & space waste)
- Photo (not standard in US/Canada, can introduce bias)
- Age, marital status, or other personal info

---

## 8. PROFESSIONAL SUMMARY/OBJECTIVE (Weight: 5%)

### When to Include
- Career changers (explain transition)
- Targeting specific role type
- Highlight unique value proposition

### Format (2-3 lines maximum)
"[Years of experience] [Role] specializing in [key skills/technologies]. Proven track record of [major achievement category]. Seeking [specific role] to [clear goal aligned with company]."

**Example:**
"Software Engineer with 3 years of experience building scalable web applications using React and Node.js. Delivered features used by 2M+ users at Samsung. Seeking full-stack role at innovative fintech company."

### Avoid
- Generic statements ("hardworking team player")
- Vague aspirations ("seeking challenging opportunity")
- Longer than 3 lines
- Include only if it adds clear value

---

## 9. ADDITIONAL SECTIONS (Weight: 5%)

### Awards & Recognition
- Industry awards
- Company recognition
- Hackathon wins (with placement/details)
- Academic honors (if recent grad)

### Certifications
- Professional certifications (AWS, GCP, PMP, etc.)
- Include expiration dates if relevant
- Only include current/recent certifications

### Publications & Patents
- For research/senior roles
- Include links if accessible
- Co-author status acceptable

### Leadership & Extracurriculars
- Leadership roles (club president, team lead)
- Volunteer work (if demonstrates relevant skills)
- Less important for experienced roles
- 2-3 items maximum

---

## 10. OVERALL QUALITY CHECKS (Weight: 5%)

### Grammar & Spelling
- Zero spelling errors (critical failure point)
- Zero grammatical errors
- Consistent tense usage
- No typos in company names or technologies

### Consistency
- All bullets use same format (period or no period - choose one)
- Date formats consistent throughout
- Font sizes consistent
- Bold/italic usage consistent
- Spacing between sections consistent

### Professionalism
- No casual language or slang
- No humor or emojis
- No first-person pronouns ("I", "my")
- Professional tone throughout

### Attention to Detail
- Company names spelled correctly
- Technology names spelled correctly
- Numbers formatted consistently (1,000 vs 1000)
- URLs work and are current
- No broken LinkedIn/GitHub links

## SPECIAL CRITERIA FOR FAANG/TOP TECH COMPANIES

### Google's 4 Core Attributes
1. **Role-Related Knowledge (RRK)** - Technical skills & domain expertise
2. **General Cognitive Ability (GCA)** - Problem-solving demonstrated through achievements
3. **Leadership** - Initiative, impact beyond individual role, mentorship
4. **Googleyness** - Collaboration, learning from failure, ambitious thinking

### Key Success Factors for FAANG
- **6-Second Rule:** Most critical info scannable in 6 seconds
  - Last position & company
  - Total years of experience
  - Education credentials
  - Top 2-3 technical skills

- **Demonstrate Scale:**
  - User numbers
  - Data volumes
  - Team sizes
  - Revenue impact
  - Performance metrics

- **Show Ownership:**
  - "Led" vs "participated in"
  - Individual contributions clear
  - End-to-end project ownership
  - Drove outcomes, not just completed tasks

- **Technical Depth:**
  - System design experience
  - Architecture decisions
  - Trade-off considerations
  - Production-level work

## RED FLAGS TO AVOID

### Content Red Flags
❌ Employment gaps > 6 months without explanation
❌ Job hopping (< 1 year at multiple companies) without context
❌ Vague responsibilities without achievements
❌ Irrelevant work experience taking up space
❌ Outdated skills (Flash, FrontPage, etc.)
❌ Lies or exaggerations (easily caught)

### Format Red Flags
❌ Longer than 2 pages
❌ Tiny font (< 10pt) or huge font (> 12pt)
❌ Walls of text without bullets
❌ Inconsistent formatting
❌ Multiple columns
❌ Colored backgrounds or graphics

### Language Red Flags
❌ Passive voice ("was responsible for")
❌ Cliches ("team player," "detail-oriented," "think outside the box")
❌ Jargon without context
❌ Spelling/grammar errors
❌ Informal language

## IMPLEMENTATION GUIDE FOR AI FEEDBACK

### Evaluation Process
1. **Parse Resume:** Extract all sections and information
2. **Score Each Category:** Use 1-100 scale per section
3. **Calculate Weighted Total:** Apply weights based on experience level
4. **Generate Specific Feedback:** Not just scores, but actionable improvements
5. **Prioritize Issues:** Flag critical errors first, then improvements

### Feedback Format
For each issue identified, provide:
- **Category:** Which section/criteria
- **Severity:** Critical / High / Medium / Low
- **Issue:** What's wrong
- **Impact:** How it affects their application
- **Fix:** Specific actionable recommendation
- **Example:** Before/after if applicable

### Sample Feedback Structure
```
OVERALL SCORE: 72/100 - Fair Match

CRITICAL ISSUES (Fix Immediately):
1. ATS Compatibility: Resume uses 2-column layout that ATS cannot parse correctly
   → Fix: Convert to single-column format
   → Impact: Current format may cause 70% rejection rate

2. Missing Quantifiable Results: Only 2 of 12 bullets include metrics
   → Fix: Add numbers to at least 8 bullets (67% target)
   → Example: "Improved performance" → "Improved page load time by 45% (2.1s to 1.2s)"

HIGH PRIORITY (Address Soon):
3. Keyword Match: Only 58% match with target job description
   → Missing: React Native, GraphQL, CI/CD, Agile
   → Fix: Add these technologies to skills section and relevant experience bullets

RECOMMENDATIONS (Nice to Have):
4. Projects Section: Add GitHub links to both projects
5. Leadership: Highlight team lead experience more prominently
```

## ADDITIONAL NOTES

### Resume Tailoring Requirement
- Every job application needs a customized resume
- Change: keywords, skills emphasis, project selection, bullet priorities
- No "one-size-fits-all" resume for competitive roles

### ATS vs Human Review
- Resume must pass BOTH ATS scanning AND human review
- ATS focuses on: keywords, format, parsing
- Humans focus on: impact, story, culture fit
- Optimize for both audiences

### Common Myths Debunked
- ❌ Myth: ATS auto-rejects resumes → Reality: Humans make final decision
- ❌ Myth: More keywords = better → Reality: Natural usage matters more than quantity
- ❌ Myth: Creative design stands out → Reality: Simple format works best
- ❌ Myth: 100% keyword match is ideal → Reality: 70-85% is the sweet spot
